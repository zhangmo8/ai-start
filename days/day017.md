## 核心概念笔记：Shape 和 Broadcasting

### 一、Shape 是什么

**Shape** 是 NumPy 用来描述张量"长什么样"的方式——它是一个整数元组，表示每个维度上有多少个元素。

```
a = np.array([[1, 2, 3],    # shape = (2, 3)  → 2行3列的矩阵
              [4, 5, 6]])

b = np.array([1, 2, 3])     # shape = (3,)   → 3个元素的一维向量

c = np.array([[[1, 2],      # shape = (2, 2, 2) → 2个2×2矩阵堆叠
               [3, 4]],
              [[5, 6],
               [7, 8]]])
```

**Shape 的核心作用：**
- 告诉程序员这个张量有几维、每维多大
- 是判断两个张量能否做运算的关键依据
- 在模型中决定权重矩阵的形状、决定数据流动的尺寸

### 二、Broadcasting（广播）是什么

**Broadcasting** 是 NumPy 处理「形状不同的数组之间做运算」的规则——它会自动把较小的数组"拉伸"成和较大的数组匹配的形状，而不需要真正复制数据。

**Broadcasting 规则（从右向左对齐）：**
1. 从右边开始比较两个 shape
2. 如果维度大小相同，或者其中一个为 1，则兼容
3. 如果维度大小不同且都不为 1，则不兼容
4. 兼容的情况下，结果 shape 是每个维度取最大值

**常见示例：**

```
A: shape (3, 4)    +    B: shape (4,)    →    result: shape (3, 4)
                     ↓
              自动复制成 (4,) → (1, 4) → 再复制成 (3, 4)

A: shape (3, 1)    +    B: shape (1, 4)    →    result: shape (3, 4)
                     ↓
              A 从 (3,1) 广播成 (3,4)
              B 从 (1,4) 广播成 (3,4)
```

**图示：Broadcasting 过程**

```
输入 A                    输入 B                  广播后
shape (3,1)              shape (1,4)            ----------
[ [1] ]                  [ [1, 2, 3, 4] ]       [ [1, 2, 3, 4] ]
[ [2] ]    +  broadcast → [ [1, 2, 3, 4] ]  =  [ [2, 4, 6, 8] ]
[ [3] ]                  [ [1, 2, 3, 4] ]       [ [3, 6, 9, 12] ]
                                                     ↓
                                                 输出 shape
                                                     (3, 4)
```

**联系：** 
- 向量/矩阵/张量是"物体"，shape 是描述它们的"尺子"
- 当两个张量形状不同时，broadcasting 决定能否以及如何运算
- 三者组合起来，才能完整描述神经网络中的数据流动

### 四、常见误区

1. **Shape 不同就报错？** 不一定——满足 broadcasting 规则就能运算
2. **Broadcasting 会复制数据？** 不会——只是逻辑上的拉伸，底层是向量化操作
3. **所有不匹配的 shape 都能广播？** 否——如 (3,4) 和 (5,6) 无法广播

## 自查问题回答

**1. 今天学的 `shape` 和 `broadcasting`，在「从 0 做模型」的流程中会用在哪些具体环节？**
- 数据预处理：把不同形状的输入统一成 batch
- 权重与激活值运算：矩阵乘法、自动广播求导
- 损失函数：预测值与标签的形状匹配
- 构建网络层：Embedding lookup、Attention score 计算

**2. 如果两个数组 shape 不匹配，NumPy 会如何处理？可能导致什么隐藏 bug？**
- 如果不满足 broadcasting 规则，NumPy 会抛出 `ValueError: operands could not be broadcast together`
- 隐藏 bug：有时 broadcasting 会产生意外结果——如 (3,4) + (4,) 被理解成 (3,4) + (1,4) 再广播成 (3,4)，但程序员可能误以为是 (3,4) + (4,1)

**3. 对比手写代码实现广播 vs 使用 NumPy 内置广播，各有什么优劣？**
- 手写广播：灵活可控，但易出错，性能差
- NumPy 广播：高性能、向量化、无需复制数据，但规则不直观可能产生 bug
