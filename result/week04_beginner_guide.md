## 先把几个词翻成大白话

### logits

`logits` 可以先理解成：

> 模型对每个候选答案打的原始分数

它们不是概率。

特点是：

- 可以是正数
- 可以是负数
- 可以很大
- 也可以很小

例如：

```python
[2.0, 1.0, 0.1]
```

这只是三个分数。

### 概率分布

概率分布要求：

- 每个值都在 `0` 到 `1` 之间
- 所有值加起来等于 `1`

例如：

```python
[0.7, 0.2, 0.1]
```

这才是概率分布。

### softmax

softmax 的作用是：

> 把一组原始分数变成概率分布

### cross entropy

交叉熵可以先理解成：

> “你给正确答案分了多少概率”

如果正确答案概率很高，loss 就小。

如果正确答案概率很低，loss 就大。

### gradient / 梯度

梯度就是：

> 告诉你往哪个方向改，loss 会下降

你可以先把它想成“坡度”。

## Day 023：logits 与概率分布

这一天天然是“先分清分数和概率”。

### logits 是什么

假设词表里有 3 个候选答案：

- A
- B
- C

模型先给它们打分：

```python
[2.0, 1.0, 0.1]
```

这表示：

- A 分最高
- B 次之
- C 最低

但这不是概率。

### 为什么不能直接把 logits 当概率

因为概率必须满足：

- 都在 0 到 1 之间
- 总和等于 1

而 logits 没有这个要求。

比如：

```python
[10, -3, 7]
```

这明显不是概率，但它可以是模型的原始打分。

### logits 最重要的性质

只要分数排序不变，模型最关心的“谁最大”其实没变。

比如：

```python
[2.0, 1.0, 0.1]
```

和

```python
[20.0, 19.0, 18.1]
```

排序都一样。

这就是为什么 logits 很适合做“中间打分”。

### 你该问的问题

- “这里是 logits 还是概率？”
- “这个数能不能直接解释成概率？”
- “谁是最大分数？”

## Day 024：softmax

这一天天然是“把分数变成概率”。

### softmax 在做什么

softmax 会把一组分数变成概率。

比如 logits 是：

```python
[2.0, 1.0, 0.1]
```

softmax 之后可能变成：

```python
[0.66, 0.24, 0.10]
```

你可以把它理解成：

- A 有 66% 的把握
- B 有 24% 的把握
- C 有 10% 的把握

### softmax 为什么有用

因为它把“谁更大”的比较，变成了“可解释的概率”。

### softmax 的两个关键特点

1. 输出一定是正数
2. 所有输出加起来等于 1

### 为什么要先减最大值

工程上常见的稳定写法，会先减去最大值，避免数太大。

这不是数学花活，而是防止计算爆掉。

### softmax 最值得记住的一点

logits 里谁最大，softmax 之后通常谁还是最大。

也就是说：

> softmax 不会改变“谁最强”这个排序直觉

它只是把分数变得像概率。

### 你该问的问题

- “softmax 之后为什么和是 1？”
- “为什么要先减最大值？”
- “softmax 之前和之后，shape 变了吗？”

## Day 025：交叉熵

这一天天然是“看正确答案有多惨”。

### 交叉熵到底在量什么

如果正确答案是 A，而模型只给 A 很小的概率，那么它就“很错”。

交叉熵就是用来衡量这个“很错”有多严重。

### 一个最简单的理解

假设真实答案是 A：

- 如果模型给 A 的概率是 `0.9`，loss 就小
- 如果模型给 A 的概率是 `0.1`，loss 就大

### 为什么是 `-log(p_true)`

因为：

- `p_true` 越接近 `1`，loss 越接近 `0`
- `p_true` 越接近 `0`，loss 越大

这正好符合“预测得越差，惩罚越大”。

### 为什么不用简单准确率

因为准确率太粗糙了。

它只告诉你：

- 对了
- 错了

但不知道“错得有多严重”。

交叉熵能告诉模型：

- 这次只是差一点
- 还是差得离谱

这就是它适合训练的原因。

### 你该问的问题

- “正确答案的概率是多少？”
- “为什么概率越小，loss 越大？”
- “这个 loss 是在惩罚什么？”

## Day 026：精读资料和 autograd

这一天天然是“看 PyTorch 怎么把这些概念落地”。

### autograd 是什么

autograd 可以先理解成：

> 自动帮你算梯度的系统

你不需要自己手算每一步导数，它会追踪你做过的运算，并帮你把梯度算出来。

### 你可以把训练想成三步

1. 前向传播：算出预测和 loss
2. 反向传播：算出每个参数的梯度
3. 参数更新：沿着让 loss 变小的方向改参数

### 为什么这很重要

因为真正训练模型时，你不是只想知道“结果错了”，而是想知道：

> 到底该改哪个参数，改多少，才会变好

这就是梯度的价值。

### 你该问的问题

- “梯度到底在告诉我什么？”
- “loss 变大和变小，参数是怎么动的？”
- “forward 和 backward 各在干什么？”

## Day 027：为什么 next-token prediction 用交叉熵

这一天天然是“把前面所有概念串起来”。

### next-token prediction 是什么

给你一句话前半段：

```text
I like eating
```

模型要预测下一个 token 是什么。

这本质上是一个分类问题：

- 候选答案是词表里的每个 token
- 模型要从很多类里面选一个

### 为什么用交叉熵

因为交叉熵非常适合“多分类”。

模型输出的是每个 token 的概率分布：

- 正确 token 概率高，loss 小
- 正确 token 概率低，loss 大

这正好和训练目标一致。

### 为什么不是直接比“对不对”

因为模型需要连续、平滑、可微的反馈。

“对/错”太硬了，不适合梯度下降。

交叉熵提供的是：

> 不是只告诉你错了，而是告诉你错得有多离谱

- “词表里每个 token 都是一个类吗？”
- “为什么 cross entropy 比 accuracy 更适合训练？”
