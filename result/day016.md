# Day 016 学习成果：向量、矩阵、张量

## 核心概念解释

### 1. 向量（Vector）
- **定义**：一维的数值序列，是线性代数最基本的单位
- **形状特征**：只有1个维度，比如 `(n,)`
- **作用场景**：表示单个样本的特征、模型的单个嵌入向量、单个预测输出
- **NumPy 对应**：`np.array([1, 2, 3])` → shape (3,)
- **模型中的用途**：比如大模型里单个 token 的 embedding 就是一个向量

### 2. 矩阵（Matrix）
- **定义**：二维的数值表格，由行和列组成
- **形状特征**：有2个维度，比如 `(m, n)` 表示m行n列
- **作用场景**：表示多个样本的特征集合、线性变换的参数、注意力权重矩阵
- **NumPy 对应**：`np.array([[1, 2, 3], [4, 5, 6]])` → shape (2, 3)
- **模型中的用途**：比如一批10个样本的embedding就是(10, 768)的矩阵

### 3. 张量（Tensor）
- **定义**：n维的数值容器，是向量和矩阵的泛化
- **形状特征**：可以有任意多维度，0维是标量，1维是向量，2维是矩阵，≥3维统称张量
- **作用场景**：表示批量的高维数据，比如图像、视频、批量文本序列
- **NumPy 对应**：`np.random.rand(32, 128, 768)` → shape (32, 128, 768)（32个样本，每个128个token，每个token768维）
- **模型中的用途**：Transformer 输入的批量数据几乎都是三维及以上的张量

### 三者的关系
```
标量（0维）→ 向量（1维）→ 矩阵（2维）→ 张量（≥3维）
└───────────────────────────────────────────┘
         本质上都是不同维度的张量
```

## 自测判断题

1. **[ ]** 形状为(100,)的数组是一个矩阵
2. **[ ]** 大模型中输入的一批32条、每条128个token、每个token768维的embedding，是一个形状为(32, 128, 768)的张量
3. **[ ]** NumPy 的 ndarray 只能表示三维及以上的张量

### 答案
1. ❌ 一维数组是向量，矩阵必须是二维
2. ✅ 符合三维张量的定义
3. ❌ ndarray 可以表示任意维度的数值数组，包括标量、向量、矩阵

## 与模型工程主线的关联
这些概念是所有深度学习框架的基础：
- PyTorch 的 Tensor 就是对这些概念的工程实现
- 模型的所有输入输出、参数权重都是张量
- 理解 shape 是调试模型维度不匹配错误的前提
