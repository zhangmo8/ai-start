# Day 017 补充讲义：shape 和 broadcasting

## 先说结论

今天你要学会两件事：

1. `shape` 是数组的“地图”
2. `broadcasting` 是 NumPy 的“自动对齐规则”

---

## 先理解 shape

`shape` 说的就是：

> 每一维有多长

比如：

```python
scores = np.array([
    [90, 85, 88],
    [76, 92, 81]
])

print(scores.shape)  # (2, 3)
```

你不要只记 `(2, 3)`。

你要翻译成：

- 2 个学生
- 每个学生 3 门成绩

所以 `shape` 不是数字游戏，而是“数据在现实里怎么摆放”的说明书。

---

## 再理解 broadcasting

广播的意思是：

> 两个 shape 不一样的数组，在满足规则时，也能做逐元素运算

比如：

```python
scores = np.array([
    [90, 85, 88],
    [76, 92, 81]
])

bonus = np.array([5, 0, 2])
```

虽然 `scores.shape = (2, 3)`，`bonus.shape = (3,)`，但它们可以相加：

```python
scores + bonus
```

这时 NumPy 会把 `bonus` 理解成：

```text
[
  [5, 0, 2],
  [5, 0, 2]
]
```

意思就是：

> 同一组加分，给每一行都加一遍

---

## 广播的最简单规则

从最后一维往前看：

- 一样大，可以对齐
- 其中一个是 `1`，可以扩展
- 其他情况不行

例子：

- `(2, 3)` 和 `(3,)` 可以
- `(2, 3)` 和 `(1, 3)` 可以
- `(2, 3)` 和 `(2, 1)` 可以
- `(2, 3)` 和 `(2,)` 不行

---

## 你可以把广播想成什么

你可以把它想成：

> NumPy 帮你把小数组“复制到合适的位置”，再做逐元素计算

它不是乱补，而是按规则补。

---

## 最小 demo

```python
import numpy as np

scores = np.array([[90, 85, 88], [76, 92, 81]])
bonus_per_subject = np.array([5, 0, 2])
extra_per_student = np.array([[3], [1]])

print(scores + bonus_per_subject)
print(scores + extra_per_student)
```

你要自己先猜：

- 第一个是按哪一维加？
- 第二个是按哪一维加？

---

## 常见误区

### 误区 1：广播就是自动瞎补

不对。广播有规则。

### 误区 2：只要维度不同就不能算

不对。有些 shape 不一样也能对齐。

### 误区 3：广播会改变原数组 shape

通常不会。它更多是在运算时临时对齐。

---

## 你什么时候该问问题

你只要卡在这些地方，就该问：

- “这个 shape 为什么能相加？”
- “它是沿着哪一维广播的？”
- “为什么这个能算，那个不行？”
- “这里补的是哪一个维度？”

可以直接问：

- “这个广播是按行还是按列？”
- “为什么 `(2, 3)` 和 `(2,)` 不行？”
- “这个 `1` 是怎么起作用的？”
