## 先别怕：这些词都可以翻译成人话

### 标量 scalar

标量就是一个单独的数。

比如：

```python
7
```

你可以把它理解成：

- 没有行
- 没有列
- 没有多层
- 只有一个值

它是 `0` 维。

### 向量 vector

向量就是一排数字。

比如：

```python
[10, 20, 30]
```

你可以把它理解成一个列表，或者一维数组。

它的 `shape` 是 `(3,)`，意思是：

- 只有 1 个维度
- 这一维长度是 3

### 矩阵 matrix

矩阵就是一个二维表格。

比如：

```text
[
  [1, 2, 3],
  [4, 5, 6]
]
```

这就是 `2` 行 `3` 列。

它的 `shape` 是 `(2, 3)`。

### 张量 tensor

张量不是神秘东西。

你可以把它理解成：

> 更高维的数组

例如一个 `shape = (2, 3, 2)` 的数组，可以理解成：

- 2 个小表格
- 每个小表格 3 行 2 列

也就是说，张量只是“层数更多的数组”。

## 你应该怎样读 shape

这是这周最重要的习惯。

看到一个数组，先别急着算，先问：

> 每一维代表谁？

比如：

```python
scores.shape == (2, 3)
```

你不要只把它记成“2 和 3”。

你应该试着翻译成：

- 2 个对象
- 每个对象有 3 个特征

如果这个数组表示成绩，就可以读成：

- 2 个学生
- 每个学生 3 门课

如果它表示模型中的数据，也可能读成：

- 2 条样本
- 每条样本 3 个特征

所以 `shape` 不是装饰品，而是阅读数组的地图。

## Day 016: 向量、矩阵、张量

这一天天然适合做“定义对照表”。

### 你要记住什么

- 向量：一排数
- 矩阵：一个表
- 张量：更多层的表

### 你真正要理解什么

它们不是三个完全不同的东西，而是同一种东西在不同维度下的名字。

### 你可以问自己什么

- 这个数据是几维？
- 每一维表示什么？
- 这些数字是按什么方式组织的？

### 常见误解

- 误解 1：张量一定很复杂
- 事实：张量只是更高维的数组

- 误解 2：矩阵和表格无关
- 事实：矩阵最容易先用“表格”去理解

### 如果你卡住，应该问什么

- “这个 shape 怎么读成现实里的意思？”
- “为什么这个数组叫向量，不叫普通列表？”
- “这个 3 维数组每一层代表什么？”

## Day 017: shape 和 broadcasting

这一天天然是“看 shape 的能力训练”。

### shape 是什么

shape 就是每一维有多长。

比如：

```python
[1, 2, 3].shape  # (3,)
[[1, 2, 3], [4, 5, 6]].shape  # (2, 3)
```

### broadcasting 是什么

广播就是：

> 两个 shape 不完全一样的数组，在满足规则时也能做逐元素运算

比如：

```python
scores = [[90, 85, 88],
          [76, 92, 81]]

bonus = [5, 0, 2]
```

你可以把它理解为：

- `bonus` 不是只给一行加分
- 而是被自动复制到每一行

### 你要记住的规则

从最后一维往前看：

- 一样大，可以对齐
- 其中一个是 `1`，可以扩展
- 否则不行

### 你应该怎么想

不要把广播当成“魔法”。

把它当成：

> “NumPy 按规则自动帮你重复某些维度”

### 你该问的问题

- “它是沿着哪一维广播的？”
- “补齐的是哪一个维度？”
- “为什么这个能加，那个不能加？”

## Day 018: 矩阵乘法与维度变化

这一天天然是“看懂变换”。

### 矩阵乘法不是逐元素相乘

这点非常关键。

```python
a * b
```

通常表示位置对位置相乘。

```python
a @ b
```

表示矩阵乘法。

### 为什么矩阵乘法重要

因为它会把输入特征变成输出特征。

例如：

- 输入有 3 个特征
- 经过权重矩阵后，变成 4 个特征

这就是模型里最常见的“线性层”思路。

### 记公式时不要死背

你只需要先记住：

```text
(2, 3) @ (3, 4) -> (2, 4)
```

意思是：

- 左边保留 2
- 中间的 3 要对上
- 右边保留 4

### 你该问的问题

- “为什么中间那个维度必须对齐？”
- “输出 shape 为什么是这个？”
- “这个运算是在混合什么特征？”

## Day 020: 整合与表达

这一天天然是“你能不能讲给别人听”。

### 你需要会说的不是术语，而是串联

你应该能把这几件事连起来：

- 数据可以被组织成向量、矩阵、张量
- `shape` 用来描述结构
- broadcasting 解决逐元素对齐
- 矩阵乘法负责特征变换
- 这些东西共同构成模型里的数据流

### 这天最重要的练习

你要试着画图。

比如：

```text
输入 x:  (B, C)
权重 w:  (C, H)
输出 y:  (B, H)
偏置 b:  (H,)
```

然后解释：

- `B` 是批次
- `C` 是输入特征
- `H` 是输出特征

## 你什么时候应该提问

你不用等到“完全不会”才问。

你应该在下面这些时刻问：

1. 你能看懂公式，但看不懂 shape。
2. 你能跑代码，但不知道为什么能跑。
3. 你能看到输出，但解释不了每一维代表什么。
4. 你知道广播发生了，但不知道补的是哪一维。
5. 你知道矩阵乘法结果变了，但不知道为什么中间维会被“消耗”。

### 可以直接这样问

- “这个 shape 应该怎么翻译成人话？”
