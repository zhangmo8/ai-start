# Day 023 补充讲义：logits 与概率分布

## 先说结论

今天只学一件事：

> logits 是原始分数，概率分布才是“能解释成概率”的结果

---

## logits 是什么

你可以先把 logits 看成：

> 模型给每个候选答案打的分

比如：

```python
[2.0, 1.0, 0.1]
```

这意味着：

- 第 1 个最强
- 第 2 个次之
- 第 3 个最弱

但这还不是概率。

---

## 概率分布是什么

概率分布要求：

- 每个数都在 0 到 1 之间
- 所有数加起来等于 1

比如：

```python
[0.66, 0.24, 0.10]
```

这才像概率。

---

## 为什么不能直接把 logits 当概率

因为 logits 可能是：

- 正数
- 负数
- 很大
- 很小

例如：

```python
[10, -3, 7]
```

这明显不能直接解释成概率。

---

## 为什么模型喜欢输出 logits

因为 logits 是“原始打分”，比较灵活。

模型先把每个候选都打分，再交给 softmax 去变成概率。

你可以把 logits 想成：

> 先排顺序，再算概率

---

## 最小 demo

```python
import numpy as np

logits = np.array([2.0, 1.0, 0.1])
print(np.argmax(logits))
```

你可以先问自己：

- 谁分最高？
- 这个最高分的位置代表什么？

---

## 常见误区

### 误区 1：logits 就是概率

不对。

### 误区 2：logits 必须正数

不对。它可以有负数。

### 误区 3：logits 和概率没有关系

也不对。它们是前后关系，softmax 会把 logits 变成概率。

---

## 你什么时候该问问题

- “这个分数能不能直接解释成概率？”
- “为什么模型先输出分数，不直接输出概率？”
