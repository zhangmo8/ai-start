# Day 018 补充讲义：矩阵乘法与维度变化

## 先说结论

今天只要抓住一句话：

> 矩阵乘法不是“每个数相乘”，而是“把输入特征变成输出特征”。

---

## 先分清楚两种乘法

### `*`

通常表示逐元素相乘。

比如：

```python
a * b
```

意思是：

- 位置对位置算
- 第 1 个乘第 1 个
- 第 2 个乘第 2 个

### `@`

表示矩阵乘法。

比如：

```python
a @ b
```

它不是简单把对应位置乘一下，而是用“行和列”的组合来算。

---

## 为什么矩阵乘法重要

因为模型里最常见的事情就是：

> 输入有一组特征，经过权重变换后，输出另一组特征

例如：

```python
x.shape = (2, 3)
w.shape = (3, 4)
y = x @ w
```

结果是：

```python
y.shape = (2, 4)
```

这句话的意思是：

- 2 条样本保留
- 每条样本原来有 3 个特征
- 经过变换后变成 4 个特征

---

## 为什么中间那个维度必须对上

因为矩阵乘法里，左边最后一维和右边第一维要匹配。

所以：

```text
(2, 3) @ (3, 4) -> (2, 4)
```

这里的 `3` 和 `3` 必须对齐。

你可以把这个 `3` 理解成：

> 被“消费掉”的维度

---

## 这和模型有什么关系

线性层就是这个思路：

- `x` 是输入
- `w` 是权重
- `b` 是偏置
- 输出是新的特征表示

比如：

```python
y = x @ w + b
```

其中 `b` 往往会广播到每一行。

---

## 最小 demo

```python
import numpy as np

x = np.array([[1, 2, 3], [4, 5, 6]])
w = np.array([[1, 0], [0, 1], [1, 1]])
b = np.array([0.1, 0.1])

y = x @ w + b
print(y)
print(y.shape)
```

你先自己猜：

- 输出 shape 是多少？
- 为什么偏置 `b` 能加上去？

---

## 常见误区

### 误区 1：`*` 和 `@` 差不多

不对。它们干的是完全不同的事。

### 误区 2：矩阵乘法只是数学题

不对。它就是模型里的特征变换。

### 误区 3：输出 shape 随便变

不对。输出 shape 是有规则的。

---

- “为什么输出 shape 是这个？”

- “这个运算是在混合什么特征？”

可以直接问：

- “这里是线性变换吗？”
- “输入维和输出维分别是什么？”
